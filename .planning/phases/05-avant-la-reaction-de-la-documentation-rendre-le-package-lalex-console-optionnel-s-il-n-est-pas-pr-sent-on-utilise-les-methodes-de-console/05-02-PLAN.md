    **Step 4 — Add log() helper function (D-02, D-05):**
    After `let currentLogLevel`, add:
    ```typescript
    /**
     * Posts a log message to the client thread if the message level passes
     * the current threshold (worker-side filtering — D-05).
     * The client's onmessage handler dispatches it through LL (real or shim).
     */
    function log(level: LogLevel, scope: string, ...args: unknown[]): void {
      if (shouldLog(level, currentLogLevel)) {
        self.postMessage({ type: 'log', level, scope, args });
      }
    }
    ```

    **Step 5 — Replace ALL LL.* call sites with log() calls:**

    Replace each `LL.X(message, ...)` with `log('X', \`sqlite/worker ${index + 1}\`, message, ...)`.
    The scope string format is: `sqlite/worker ${index + 1}` (per Research Open Questions #2).

    For lines within `open()` function where `index` is the parameter:
    - `LL.debug(msg)` → `log('debug', \`sqlite/worker ${index + 1}\`, msg)`
    - `LL.error(msg)` → `log('error', \`sqlite/worker ${index + 1}\`, msg)`
    - `LL.verb(msg)` → `log('verb', \`sqlite/worker ${index + 1}\`, msg)`
    - `LL.warn(msg, e)` → `log('warn', \`sqlite/worker ${index + 1}\`, msg, e)`
    - `LL.wth(msg, ...)` → `log('wth', \`sqlite/worker ${index + 1}\`, msg, ...)`

    For calls inside the query handler (also inside `open()`, `index` is still in scope):
    - Same scope pattern: `\`sqlite/worker ${index + 1}\``

    **Step 6 — Replace LL.level setter with currentLogLevel assignment (D-06):**
    In the top-level `self.onmessage` handler (near line 295):
    - Replace: `LL.level = logLevel ?? 'warn';`
    - With: `currentLogLevel = logLevel ?? 'warn';`

    **CRITICAL: Do NOT remove any log message content (the strings/args passed to LL.*).
    Only replace the LL.X(...) call with log('X', scope, ...).
    The [Worker N] prefix that was embedded in strings can be removed from message args
    since it is now captured in the scope field. Example:
    - Old: `LL.debug(\`[Worker ${index + 1}] open() called with file:\`, file)`
    - New: `log('debug', \`sqlite/worker ${index + 1}\`, 'open() called with file:', file)`
    The "[Worker N]" is dropped from the message string since scope carries that info.**

    Verify the import of `WorkerMessageData` already exists in `src/worker.ts` — if not, add it. The `log()` helper posts `{ type: 'log', ... }` which TypeScript must type-check against `WorkerMessageData`.
