    Make the following changes to `src/worker.ts`:

    **1. Remove the hardcoded LL.level (BUG-05 prerequisite)**
    At lines 11-13:
    ```typescript
    const LL = Logger.scope('sqlite/worker');
    LL.level = 'debug';
    LL.date = LL.level === 'debug';
    ```
    Remove `LL.level = 'debug';`. Also fix `LL.date` — it was `true` only when level was `'debug'`, but now level is dynamic. Replace with:
    ```typescript
    const LL = Logger.scope('sqlite/worker');
    LL.date = true;
    ```
    The `LL.level` will be set after the open message arrives (see change 3 below).

    **2. Remove the dead `log` lambda and its commented-out assignment (TYPE-02)**
    Lines 55-57 contain a commented-out lambda:
    ```typescript
    // let log = (...args: Parameters<typeof console.log>) => {
    // 	LL.wth(...args);
    // };
    ```
    Lines 65-67 contain a commented-out reassignment:
    ```typescript
    // log = (...args: Parameters<typeof console.log>) => {
    // 	LL.debug(`Worker ${index + 1}`, ...args);
    // };
    ```
    Delete both blocks entirely (remove the lines). Do not leave blank lines in their place.

    **3. Fix inverted allQueryPragmas condition (BUG-01)**
    Lines 79-83 currently read:
    ```typescript
    const allQueryPragmas = Object.keys(pragmas).length
      ? ''
      : Object.entries(pragmas)
          .map(([key, val]) => `PRAGMA ${key}=${val};`)
          .join('');
    ```
    The condition is inverted — it builds the empty string `''` when pragmas exist (`.length` is truthy), and builds the PRAGMA string when pragmas is empty (`.length` is 0/falsy). Fix by negating the condition:
    ```typescript
    const allQueryPragmas = !Object.keys(pragmas).length
      ? ''
      : Object.entries(pragmas)
          .map(([key, val]) => `PRAGMA ${key}=${val};`)
          .join('');
    ```
    Now: when `pragmas` is empty (`!Object.keys(pragmas).length` is true) → `''`, when `pragmas` is non-empty (`!Object.keys(pragmas).length` is false) → the PRAGMA string. This is the correct behavior.

    **4. Apply logLevel in the open message handler (BUG-05)**
    The outer `self.onmessage` at lines 209-215 currently destructures:
    ```typescript
    self.onmessage = async (event: MessageEvent<ClientMessageData>) => {
      const { data } = event;
      if (data.type === 'open') {
        const { file, flags, index, vfs, pragmas } = data;
        open(file, flags, index, { vfs, pragmas });
      }
    };
    ```
    Add `logLevel` to the destructure and apply it to `LL` before calling `open`:
    ```typescript
    self.onmessage = async (event: MessageEvent<ClientMessageData>) => {
      const { data } = event;
      if (data.type === 'open') {
        const { file, flags, index, vfs, pragmas, logLevel } = data;
        LL.level = logLevel ?? 'warn';
        open(file, flags, index, { vfs, pragmas });
      }
    };
    ```

    No other changes to `worker.ts` in this plan. The `@ts-expect-error` directives are handled in Plan 04.
